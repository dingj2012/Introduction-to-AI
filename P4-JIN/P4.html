Problem Goal

Atropos is two-player game invented in our own Computer Science Department by Kyle Burke and Prof. Shang-Hua Teng. For this assignment we will implement a strategy to play this game intelligently (as opposed to randomly or via brute-force search). Specifically, we are expected to

1.Design a static evaluator,

2.Select an adversarial search algorithm from the ones you have studied in class (e.g., minimax search, alpha-beta search, a heuristic pruning method),

3.Implement this algorithm and apply it to the Atropos game,

4.Clearly comment your code so that key algorithmic ideas are clear and parameters (e.g., lookahead depth of search) are easy to find and modify,

5.Package your code so that the executable can participate in an automatic tournament.

 

   Implementation 

--Parse and transfer the input:

Since Python gives strong support on type conversion, the parsing will be relatively easy: (1).board will be parsing into a list with every row of the board as string element(eg.[‘13’,’302’,’1003’,...]). (2). Last move will be transferred into the 2-D coordinate: we observed in the given(c,x,y,z), z will not be useful and c will not do much help in our evaluation strategy, thus we will ignore it first. Then (c,x,y,z) will be transferred into (size+2-x,y).

--Static evaluator :

The evaluation strategy is combined with two parts: possibleMove() and combinationCount()

possibleMove() -- One great move can make the opponent to have the least move for higher trap setting possibility. Thus the first part(possibleMove()) we evaluate a   move, we count how many free moves (how many “0”) around it.

 combinationCount() -- One great move should also avoid those “dangerous” moves. This means another point we are trying to evaluate how many colors you can put in a   specific coordinate. We doing this by evaluating how many combinations(1 and 2, 1 and 3, 2 and 3) are around the move we are going to make.

 

           Thus the evaluation point is based on a specific move, the less point is(fewer moves for opponents and fewer combinations surrounded), the better the move will be.

 

--Minimax traversing :

 

We chose Minimax to build the game tree. The tree traversing will stop if the move is leaf or the traversing reaches to the max depth. If the depth is even number, then  it will find the max throughout the leaf nodes; if the the depth is odd number, then it will find the min instead. As the depth goes higher, the algorithm will be smarter    and but slower. 

 

--helper method:

 

isLeaf() -- Determine if the move is a leaf node or not. Notice that the entire game is based on the Sperner’s Lemma which reveals there will must be at least one  triangle with its vertexes are three different colors. This means we do not need to consider the draw situation, there are always a winner and a loser.

 

pickColor() -- Based on the combinations we had, we pick a color that is available randomly(Like I said, we less consider the importance of color).

 

chooseMove() -- Do the very first minimax calculation and will set the max depth for the following minimax processes.

 

removeDup() -- remove all the duplicates in a list, used for combinationCount().
