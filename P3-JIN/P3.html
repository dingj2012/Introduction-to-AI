Problem Goal

We are required to implemente this HMM so that it handles various recognition tasks. Specifically, given the Hidden Markov model and some input sentences, we are asked to write programs that perform the following:

Pattern Recognition: Implement the "forward part" of the forward/backward procedure discussed in class. (The procedure is also described in A Tutorial on Hidden Markov Models and Selected Applications in Speech Recognition by Rabiner [1989].)   Given the HMM and several observation sequences, recognize should report the observation probability of each input sequence.

State-Path Determination: Implement the Viterbi algorithm to determine the optimal state path for each observation set and report its probability.  

Model Optimization: Optimize the HMM using the Baum-Welch algorithm and report the probabilities before and after optimization. 

 

   Implementation 

--reads input from the interface as argument:

For the three source files, we separately build up the “read” codes(“write” code also for the optimize algorithm). Three parts of code are basicly the the same with minor difference. The reading use the scan.next()...functions library (fgets() and sscanf() for the C version).

--Algorithm features:

The forward part of the recognition algorithm is pretty straight forward. Once we finished the alpha calculation, we can just do the recursive calculation. In  recognize part, the getIndex() function helps to find the index of the chosen string in the Observation Symbol list. For the Viterbi algorithm, we build both backtrack and  straight track method to keep calculating the maximum which would be a little bit violation but we couldn’t find a better way.  The Baum-Welch algorithm is also pretty  straight forward, the additional part is to implement the update sigma part which use the Bayes' theorem(hard to implement).

Answers to the questions:

--Recognize question:

I will say the lower probability is because that the actual probability is so low that  might not be represented in the sample sequence that were given. Another reason wil be the floating point problem which means part of the alpha was eliminated a few numbers. The current HMM won’t always give a reasonable answer. If the probability of a specific transition or emission is very low, the transition might never occur in the sequence states, or the emission might never occur in the sequence.In either case, the algorithm returns a probability of 0 for the given transition or emission won’t be calculated in the final P.

--Statepath question:

 

The Viterbi (partial)optimal path can be retrieved by saving back pointers that remember which state was used in the backtrack equation. Thus the Viterbi path is essentially the shortest path through the syntax analysis. The HMM cannot always correctly distinguish "statement" from "question" sentence because the “SUBJECT” and the  “OBJECT” can be mixed.

 

--Optimize question:

 

The reason we should avoid the zero observation probability is to prevent the floating point problem which caused by very little emission probability and improve the performance of the model on data not appearing in the training set.
